个人最近计划


由于我简历上的项目有点老套了，而且就从业务逻辑上来说也不是很先进
因此我想用go好好实践一下写一个AI项目
个人计划上是写一个前端简单一些的，比如微信小程序啥的
主要是当下人们的生活中，越来越多的人压力来源是个人成长或者前途之类的较为未知的压力
这些压力的处理很多时候需要一些方法论来指导，本人比较喜欢从马哲中汲取能量，所以我想做一个
方法论指导的AI项目，用RAG来构建一个基于马哲还有中国传统哲学的知识库，然后通过LLM来生成答案。

当前的技术选型上来说想的是，go后端，gin或者goframe框架，然后是LLM选qwen因为百炼的文档多一些，RAG没有实操过再看看
持久层选MongoDB，看情况添加kafka和redis

前端我是彩笔，虽然后端也是彩笔，但是前端是完全的彩笔，所以往尽量简单的整

先关注语言方面的LLM回复，在实现基础的语言回答后看情况丰富多模态这块，图片语音之类的

首先是coze来给我捋一下项目规划

以下是您提供信息的 Markdown 表格形式：

模块用途 技术选型 核心优势
用户认证与权限 MySQL + JWT 强事务支持，成熟稳定
聊天记录存储 MongoDB 6.0+ 灵活文档模型，高并发读写支持
哲学语料向量 Milvus 2.3.4 高效近似最近邻搜索，支持动态数据更新
实时通信 WebSocket (Gorilla/websocket) 低延迟双向通信，适合实时聊天
AI集成 QwenAI API + 本地Ollama嵌入模型 平衡响应速度与数据隐私
后端框架 Gin + Go 1.21+ 高性能，轻量级，适合构建RESTful API
部署环境 Docker + 阿里云ECS 容器化部署，弹性扩展
